What this page shows

Each match card compares Polymarket moneyline prices with Elo-based win/draw/loss estimates. The Edge (PP) column shows how far Elo is from the market in absolute percentage points. Strategy backtests use fixed $100 stakes on concluded matches.

Polymarket prices

Match markets come from Polymarket's Gamma API for the FIFA World Cup series (soccer-fifwc). For each match we keep the moneyline markets for home win, draw, and away win.

  • Prices are taken from Polymarket's CLOB prices-history endpoint, not from trade fills.
  • For completed matches, we average prices over the 6 hours before kickoff.
  • For upcoming matches, we average prices over the most recent 6 hours up to the current time.
  • When both YES and NO token histories exist, we average them and use the mean as the pre-match YES probability.

A Polymarket price of 0.65 is treated as a 65% implied probability for that outcome.

Elo ratings and point-in-time reconstruction

Team ratings come from eloratings.net.

  • 2026_results.tsv — completed international match results and rating updates.
  • fixtures.tsv — upcoming matches with current pre-match ratings.
  • en.teams.tsv and teams.tsv — team code lookups and name aliases.

For completed matches, each results row includes post-match Elo and the rating change for the first-listed team. Pre-match ratings are reconstructed as:

  • pre_team1_elo = post_team1_elo - rating_change
  • pre_team2_elo = post_team2_elo + rating_change

This assumes the Elo update is approximately zero-sum between the two teams, which matches the published row format. Upcoming fixtures use the pre-match ratings published directly in the fixtures file.

Estimating win, draw, and loss probabilities

Standard Elo gives win probabilities for two teams, but not draws. This page uses a Davidson-style three-outcome model:

  • Convert each team's Elo into a strength: 10 ^ (Elo / 400)
  • Add a draw strength term based on a configurable base draw rate when teams are equal
  • Normalize home win, draw, and away win probabilities so they sum to 1

Current settings: base draw probability at equal teams = 28%, home advantage = 0 Elo points (neutral-site World Cup matches).

When teams are unequal, draw probability shrinks automatically because more strength is allocated to the win outcomes.

Edge (PP)

Edge is measured in absolute percentage points:

Edge = Elo probability − Polymarket price

Example: Polymarket 0.65, Elo 0.80 → edge = +15.0 percentage points.

Positive edge means Elo thinks the outcome is more likely than the market. Edge-filtered strategies use this same absolute measure.

Data flow and exports

  • Raw downloads (Elo TSVs, Polymarket API responses) are cached locally to avoid repeated requests.
  • Derived tables — match table, strategy backtests, PnL paths — are recomputed each notebook run.
  • Notebook 06 writes site exports to docs/data/football_elo_polymarket/. API and Elo downloads cache under data/cache/ locally only.
